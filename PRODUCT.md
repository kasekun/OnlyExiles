# Product

## Register

product

## Users

Path of Exile 2 players who want to run their campaign efficiently — primarily people getting into optimization and speedrunning for the first time, not just veterans who already have routes memorized. They use this tool at a desk, before or between runs, to plan which rewards are worth stopping for and how to sequence their route. They're motivated, detail-oriented, and willing to invest time upfront to save time in-game. They appreciate being guided without being talked down to.

## Product Purpose

An interactive campaign planner for Path of Exile 2. It transforms a static reference list of Acts, Areas, and Rewards into a customizable, shareable route guide. Players mark rewards as skipped, reorder zones, annotate each area with notes and recommended levels, apply pre-built templates for common playstyles, and export or share their configuration as a clean Markdown document or unique URL. Success looks like a player finishing their planning session confident they know exactly what to pick up and where to go — and being able to hand that plan to a friend with one link.

## Brand Personality

Warm, helpful, trustworthy. Not a cold utility or a hardcore-gamer flex tool. More like a knowledgeable friend who has already mapped out the campaign and is patiently walking you through it — organized, clear, and genuinely invested in your success.

## Anti-references

- **Gaming atmosphere overload**: No elaborate textured backgrounds, glow halos, particle effects, or fantasy-font headers. The game itself handles atmosphere; the tool should step back.
- **Generic SaaS**: White backgrounds, gradient hero cards, metric dashboards, floating action buttons. This is not a productivity startup.
- **Wiki dump**: MediaWiki walls of text, unstructured tables, link-dense sidebars. The data is already complex — the UI must impose order, not add to the chaos.
- **Discord dark clone**: Mid-gray surfaces, saturated purple or blue accents, the unmistakable "server" feel.
- **Bland spreadsheet**: Dense but without intention — no visual rhythm, no hierarchy, no sense that someone cared about the experience.

## Design Principles

1. **Warm efficiency**: The tool should feel like a well-prepared friend's notes, not a cold database. Amber warmth, readable spacing, and plain language humanize an otherwise data-heavy interface.
2. **Guide, don't gate**: Treat the user as someone learning to optimize, not an expert who needs no help. Labels, placeholders, and copy explain without condescending. New users should feel oriented, not overwhelmed.
3. **Trustworthy density**: Pack in the information — PoE2 has a lot of zones and rewards — but earn the density through hierarchy and rhythm. Every typographic and color decision should make the data easier to parse, not just present.
4. **Shareable confidence**: Export and share actions should feel like publishing something valuable. The output (Markdown, shared link) should be as polished as the tool itself.
5. **Clarity over cleverness**: Interactions, labels, and layout choices should be immediately legible. No UI puzzles, no surprise interactions, no decoration that competes with the data.

## Accessibility & Inclusion

No formal WCAG level required. Aim for reasonable contrast ratios (4.5:1 for body text), full keyboard navigability for all interactive elements (skip buttons, collapse toggles, drag-to-reorder), and clear focus indicators. Color is used semantically (amber for act headers, green/blue for badge types), but not as the only indicator — shape, text, or icon always backs up color meaning. Reduced motion: collapse animations and transitions should respect `prefers-reduced-motion`.
